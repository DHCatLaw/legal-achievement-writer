# 律师业绩稿写作助手 · Legal Achievement Writer

[English](README.md) · **中文**

**一个把真实案件结果，转化为合规、贴合律所风格、可直接发布的「案件成果稿 / 业绩稿」的结构化写作助手。**

可作为可移植的 [Agent Skill](SKILL.md) 使用，也可作为 Claude Code [斜杠命令](commands/achievement.md)（`/achievement`）使用。任何律所均可配置；专为**律师**设计——尤其是必须同时满足两套执业宣传规则的**中美双重执业律师**。

---

## 为什么需要它

写一篇胜诉/成果稿，风险常被低估：同一篇文章既是**营销**，又是受规则约束的**执业宣传**。多数写作工具只为传播效果优化，会毫不犹豫地生成可能**泄密、过度承诺或诋毁对方律师**的文字。本助手把**执业道德审查放在第一位**，文采放在第二位。

### 它做什么

1. **引导式信息采集** —— 逐项问清案件类型、法院、结果、客户类别、关键数字与日期，而不是凭记忆自由发挥。
2. **强制合规审查关卡** —— 在动笔前先走一遍清单：保密（和解条款、客户同意、特权信息）、与法院文件逐一核对事实、禁止过度承诺/不当预期、禁止诋毁。详见 **[ETHICS.zh-CN.md](ETHICS.zh-CN.md)**。
3. **律所风格起草** —— 经实践检验的五段结构（开头→背景→策略→裁定→点评），并套用你所的 ALWAYS / NEVER 风格规则。
4. **按渠道优化标题与摘要** —— 在真实显示字数内优化（如微信公众号标题 ≤ 26 字、摘要 ≤ 54 字），给出多个备选而非一锤定音。
5. **干净的产出** —— 一份带日期的 Markdown 文件，内置「过往业绩不代表未来结果」免责声明与案件来源行。

### 优势

- **合规不是可选项，也不是事后补丁** —— 关卡是**拦截**不合规内容，而不是事后标记。
- **与具体律所解耦** —— 一次性初始化访谈生成 `achievement.config.md`；不内嵌任何内部路径、人名或品牌。
- **天生支持多法域** —— 保密、禁止承诺、禁止诋毁等审查同时对应美国（ABA / 各州 Rule 1.6 与 7.1）与中国（《律师法》保密义务、《律师执业行为规范》业务推广）原则；对双重执业律师，提示**从严遵守**。
- **会停下来问你** —— 律师署名与排序、每一项披露决定都先确认再写，因为这些选择带有职业含义。
- **可移植** —— 一个仓库，两种安装方式（Agent Skill 或斜杠命令）。

## 适用对象

发布案件成果的诉讼律师、知识产权 / 商事律师 —— 无论发布在律所官网、领英还是微信公众号。它**对中美双重执业律师尤其有用**：这类律师的成果稿必须在同一篇文案里同时通过两套法域的宣传与保密规则。

## 安装

**作为 Agent Skill：** 把本文件夹放到你的宿主加载 skill 的位置（它读取 `SKILL.md`）。

**作为 Claude Code 斜杠命令：**
```bash
cp commands/achievement.md ~/.claude/commands/achievement.md
# 重启 Claude Code，然后运行 /achievement
```

## 配置（一次性）

```bash
cp config.example.md achievement.config.md
# 编辑 achievement.config.md：填入律所名称、律师名册与排序政策、
# 执业法域、发布渠道与字数限制、风格 ALWAYS/NEVER、保密默认立场
```
或者：在没有配置文件时直接运行一次本助手——它会访谈你并自动写入该文件。

> 若 `achievement.config.md` 含任何非公开信息，请将其**排除在版本控制之外**。

## 使用

调用本助手（或 `/achievement`），可选地把裁定/命令 PDF 指给它。它会依次走：信息采集 → 合规关卡 → 起草 → 标题/摘要 → 产出，每一步都与你确认。

完整虚构样例见 **[examples/example-output.zh-CN.md](examples/example-output.zh-CN.md)**。
