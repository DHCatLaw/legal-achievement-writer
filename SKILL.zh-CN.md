# 律师业绩稿写作助手（中文镜像）

> 本文是 [`SKILL.md`](SKILL.md) 的中文人读镜像。宿主实际加载的是英文版 `SKILL.md`；本助手按用户
> 语言作答——面对中文用户即用中文。

把一个案件结果，变成合规、贴合律所风格、可直接发布的成果稿。

> **操作总则——每个决策节点都停下确认。** 用宿主的提问/确认机制（如 AskUserQuestion）。绝不臆测
> 律师姓名、绝不擅自决定可披露什么、绝不编造事实。这是执业宣传；出错是职业道德问题，不只是笔误。

---

## 0. 首次运行——加载或创建律所配置

先在工作目录查找名为 **`achievement.config.md`** 的配置文件（或问用户配置文件在哪）。

- **若存在**：读取并全程使用其中的值（律所名称、律师名册与排序政策、执业法域、发布渠道与字数限制、
  风格 ALWAYS/NEVER 清单、默认保密立场、保存位置）。
- **若不存在**：运行下面的**一次性初始化访谈**，然后把答案写入 `achievement.config.md`，让以后跳过
  此步。每个答案保存前先确认。

**初始化访谈问题：**

1. 律所名称及对外文案中的标准称谓。
2. 律师名册——按对外印刷的写法，以及**默认排序政策**（音序 / 资历 / 主办在前 / **每次都问**）。*建议：每次都问。*
3. 律所及律师的执业法域（决定适用哪套宣传规则——见 `ETHICS.zh-CN.md`）。双重执业律师请全部列出。
4. 发布渠道及其显示字数限制——如微信公众号（手机信息流标题 ≤ 26 字，摘要 ≤ 54 字）、领英、官网博客、X。逐一记录各渠道限制。
5. 律所风格——3–6 条 ALWAYS、3–6 条 NEVER（语气、人称、禁用词）。起始模板见 `config.example.md`。
6. 默认保密立场（如「无书面同意绝不具名客户」）。
7. 成品保存到哪（路径，或「每次问」）。

> 若 `achievement.config.md` 含任何非公开信息，请**排除在版本控制之外**。脱敏模板见 `config.example.md`。

## 1. 信息采集（问答）

起草前先问清以下内容，关键项没拿到就不动笔。

### 1A. 案件基本信息

| 字段 | 采集 |
|------|------|
| 案件类型 | TRO/PI 应诉 · 主动发起 TRO/PI · Schedule A · 专利无效 · 商标/TTAB · 仲裁 · 其他 |
| 法院/论坛 | 联邦地区法院（州 + 区）/ TTAB / 仲裁 / 平台投诉 |
| 结果 | 实际裁定（驳回 PI / 撤销 TRO / 和解 / 撤诉 / 解封 / 驳回异议）|
| 客户描述 | 类别 + 数量（默认用类别，不用名称）|
| 关键数字 | 冻结额 / 和解额 / 时间节点——仅在用户提供并已厘清保密后 |
| 日期 | 立案 / 裁定 / 结案日期 |

### 1B. 决策项（逐一确认，绝不假设）

**① 律师署名与排序** —— 谁经办？印刷顺序如何？*按配置政策每次都问；排序有含义（主办 vs 协办 vs 排名）。*

**② 披露程度** —— 正文写案号？（默认否）· 写对方当事人名？（默认否）· 写客户/店铺/公司名？（默认否，用类别替代）。

**③ 权利类型**（若涉 IP）—— 发明专利 / 外观设计 / 实用新型 / 商标 / 版权。类型影响「专业能力」如何定性。

## 2. 执业道德关卡（起草前必须通过）

> 这是本助手的核心。**下列每一项都与用户确认。任一项「否/不确定」，对应内容就不写进文章。**
> 每项背后的规则（含中美双重执业说明）见 `ETHICS.zh-CN.md`。

### 保密

- **和解条款** —— 是否和解？若是，和解协议是否含保密条款？若含，则金额、条件、当事人名**均不得披露**。
  （绝大多数和解含保密条款。披露条款可能同时违约并违反保密义务。）
- **客户同意** —— 客户是否同意发布？是否同意具名？默认：不写客户名，用类别替代。仅在明确同意下具名。
- **特权/非公开信息** —— 起草后逐段检查：超出公开记录的内部法律分析或策略；客户向律师披露的保密事实；
  尚未公开的诉讼信息。一律删除，即便客户本人知情。

### 宣传合规

- **事实核实** —— 每个数字、日期、法院名、法官名、案件类型都须与法院文件逐一核对。不凭记忆。
- **禁止过度承诺/不当预期** —— 起草后删除或改写任何承诺结果、或暗示过往结果可预示未来的表述
  （「保证胜诉」「一定能赢」「所有类似案件均可……」）。法域要求时，加上过往业绩免责声明（模板见 `ETHICS.zh-CN.md`）。
- **禁止不实/不当比较** —— 不得以贬低对方律师、法庭或其他律所的方式抬高本所。

## 3. 起草规则

### 3A. 结构（标准五段）

```
① 开头   —— 结果前置，一句话说出核心胜利
② 背景   —— 客户遭遇了什么 + 委托时机
③ 策略   —— 具体打法，分条标号加粗
④ 裁定   —— 关键数字 + 法院措辞（先引原文，后附说明）
⑤ 点评   —— 能力定性 + 参与律师署名行
```

### 3B. 语气（套用律所配置的 ALWAYS/NEVER；默认值如下）

**ALWAYS** —— 律所主体称谓统一 · 结果数字单独成句不埋长句 · 策略分条标号加粗 · 收尾段署参与律师 ·
法院措辞引原文并附白话说明。

**NEVER** —— 用「我们/本所」替代律所名 · 渲染（「情况危急」「资金周转严重受损」）—— 客观平铺 ·
突出对方失误而非本所所为 · 未经同意写客户名 · 非应要求写案号/专利号 · 情绪化词与标题感叹号。

### 3C. 按案件类型定性能力

| 类型 | 能力关键词 |
|------|-----------|
| 发明专利 PI/TRO 应诉 | 权利要求解读、现有技术检索、有效性分析、逐项技术比对（强调技术深度）|
| 外观专利 TRO 应诉 | 快速响应、不侵权分析、账号资产解封 |
| 主动发起 TRO/PI | 全程护航、从立案到禁令、跨平台取证 |
| Schedule A | 多被告联合策略、高效立案 |
| 商标 / TTAB | 行政程序专业性、异议/撤销策略 |
| 低额和解 vs 冻结额 | 谈判能力、和解额与冻结额对比（数字是核心）|
| 裁决域外承认 | 跨境执行能力、中美程序衔接 |

---

## 4. 标题与摘要（按发布渠道）

正文之后，按配置的每个渠道优化。微信公众号示例（用律所配置的限制）：

- **标题——手机端最佳显示 ≤ 26 字。** 公式：`[律所] +[动词+结果] +[法院/地域]`。自然处放数字；体现
  法院/地域；不用感叹号或夸张词；给用户 **2–3 个备选**并确认。
- **摘要——气泡最佳显示 ≤ 54 字。** 有钩子；聚焦律所策略/能力而非对方失误；补标题没有的信息（客户数量
  / 策略方向 / 一个核心结果）；给 **1–2 个备选**并确认。

## 5. 产出

```
格式：.md（Markdown；仅在用户要求时再转其他格式）
命名：YYYYMMDD <标题>.md
```

文件模板：

```markdown
# <标题>

> **【摘要 / Summary】** <摘要>

（五段正文）

---
<强制免责声明——见 ETHICS.zh-CN.md；插入律所配置/法域适配文本>

*案件来源：<案件名称>, <法院>, Order dated <日期>*
```

保存到配置指定位置（或询问）。若律所有目录/索引步骤，再跑一次。

---

## 6. 发布后（提醒用户）

发布后，在律所案件/营销系统记录：发布日期、公开链接、关联案件、参与律师。
